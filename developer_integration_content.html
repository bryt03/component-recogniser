Developer Integration Guide

Integrating the Component Recogniser into a larger web application requires a modular approach, moving beyond a single HTML file. This guide outlines common strategies and provides code snippets for popular frameworks like React and Vue.

1. Component-Based Integration (React / Vue)

For modern JavaScript frameworks, encapsulating the p5.js sketch within a dedicated UI component is the recommended approach. This allows for better state management, prop-based configuration, and event-driven communication with the rest of your application.

React Example

This example demonstrates how to create a ComponentRecognizer React component that wraps the p5.js sketch. The model URL can be passed as a prop, and classification results can be emitted via a callback prop.

import React, { useRef, useEffect, useState } from 'react';
import p5 from 'p5';
import * as ml5 from 'ml5';

const ComponentRecognizer = ({ modelUrl, onClassify }) => {
  const canvasRef = useRef(null);
  const p5Instance = useRef(null);
  const [status, setStatus] = useState('Initializing...');

  useEffect(() => {
    const sketch = (p) => {
      let classifier;
      let video;
      let flipped;
      let modelReady = false;
      let camReady = false;

      p.preload = () => {
        setStatus('Loading model...');
        try {
          classifier = ml5.imageClassifier(
            modelUrl + 'model.json',
            () => {
              modelReady = true;
              if (camReady) {
                setStatus('Ready');
                classifyFrame();
              }
            }
          );
        } catch (err) {
          setStatus('Failed to load model. Check URL.');
          console.error(err);
        }
      };

      p.setup = () => {
        const cnv = p.createCanvas(640, 480);
        cnv.parent(canvasRef.current);

        video = p.createCapture(p.VIDEO, () => {
          camReady = true;
          if (modelReady) {
            setStatus('Ready');
            classifyFrame();
          } else {
            setStatus('Camera ready - loading model...');
          }
        });
        video.size(640, 480);
        video.hide();
      };

      p.draw = () => {
        if (!camReady) return;
        flipped = ml5.flipImage(video);
        p.image(flipped, 0, 0, p.width, p.height);
      };

      const classifyFrame = () => {
        if (!classifier || !video) return;
        const src = ml5.flipImage(video);
        classifier.classify(src, (err, res) => {
          if (err) {
            console.error('Classification error:', err);
            // Optionally, re-attempt classification after a delay
            setTimeout(classifyFrame, 1000);
            return;
          }
          if (onClassify && res) {
            onClassify(res);
          }
          classifyFrame(); // Loop
        });
      };
    };

    p5Instance.current = new p5(sketch, canvasRef.current);

    return () => {
      // Cleanup: Stop the p5 sketch and webcam when component unmounts
      if (p5Instance.current) {
        p5Instance.current.remove();
      }
    };
  }, [modelUrl, onClassify]);

  return (
    <div>
      <div ref={canvasRef}></div>
      <p>Status: {status}</p>
    </div>
  );
};

export default ComponentRecognizer;

Vue Example

This Vue component demonstrates a similar integration, using onMounted and onUnmounted hooks for lifecycle management.

<template>
  <div>
    <div ref="canvasContainer"></div>
    <p>Status: {{ status }}</p>
  </div>
</template>

<script setup>
import { ref, onMounted, onUnmounted } from 'vue';
import p5 from 'p5';
import * as ml5 from 'ml5';

const props = defineProps({
  modelUrl: String,
});

const emit = defineEmits(['classify']);

const canvasContainer = ref(null);
const p5Instance = ref(null);
const status = ref('Initializing...');

onMounted(() => {
  const sketch = (p) => {
    let classifier;
    let video;
    let flipped;
    let modelReady = false;
    let camReady = false;

    p.preload = () => {
      status.value = 'Loading model...';
      try {
        classifier = ml5.imageClassifier(
          props.modelUrl + 'model.json',
          () => {
            modelReady = true;
            if (camReady) {
              status.value = 'Ready';
              classifyFrame();
            }
          }
        );
      } catch (err) {
        status.value = 'Failed to load model. Check URL.';
        console.error(err);
      }
    };

    p.setup = () => {
      const cnv = p.createCanvas(640, 480);
      cnv.parent(canvasContainer.value);

      video = p.createCapture(p.VIDEO, () => {
        camReady = true;
        if (modelReady) {
          status.value = 'Ready';
          classifyFrame();
        } else {
          status.value = 'Camera ready - loading model...';
        }
      });
      video.size(640, 480);
      video.hide();
    };

    p.draw = () => {
      if (!camReady) return;
      flipped = ml5.flipImage(video);
      p.image(flipped, 0, 0, p.width, p.height);
    };

    const classifyFrame = () => {
      if (!classifier || !video) return;
      const src = ml5.flipImage(video);
      classifier.classify(src, (err, res) => {
        if (err) {
          console.error('Classification error:', err);
          setTimeout(classifyFrame, 1000);
          return;
        }
        if (res) {
          emit('classify', res);
        }
        classifyFrame(); // Loop
      });
    };
  };

  p5Instance.value = new p5(sketch, canvasContainer.value);
});

onUnmounted(() => {
  // Cleanup: Stop the p5 sketch and webcam when component unmounts
  if (p5Instance.value) {
    p5Instance.value.remove();
  }
});
</script>

2. Headless Worker Approach

If your application doesn't require displaying the live video feed from p5.js, you can run the classification logic "headlessly." This means the p5.js canvas is not rendered, but the webcam feed is still processed by ml5.js.

  • Hidden Video Element: Instead of p.createCanvas, you might directly use a hidden <video> element in your HTML and pass it to p.createCapture.
  • Data-Only Output: The onClassify callback (or gotResult in the original sketch) would be the primary way to interact with the classification results. You would then use your application's native UI to display this data.

3. API & Backend Integration

For more complex applications, the client-side recognition can be a trigger for backend processes:

  • Database Lookup: Once a component is identified with high confidence, send the classification label to your backend API. The backend can then query a database for detailed information (e.g., datasheets, inventory, pricing) and return it to the client.
  • Audit Logging/Verification: Capture a still image from the video feed (p.get().save('snapshot.png') or canvas.toDataURL()) when a significant classification occurs. This image, along with the classification data, can be uploaded to a server for audit trails, quality control, or manual verification.

Architectural Recommendations

To maintain a clean and scalable codebase, consider separating concerns:

  • ScannerService.js (or similar): A pure JavaScript module (not tied to a UI framework) that handles the ml5.imageClassifier instantiation, webcam access, and the classification loop. It should expose methods to start/stop scanning and emit events (or use callbacks) for classification results.
  • ScannerUI.jsx/.vue: A UI component responsible solely for rendering the video feed (if needed) and displaying the classification results received from the ScannerService. It should not contain the core ML logic.
  • App.js/.vue (or parent component): Manages the overall application state, orchestrates communication between the ScannerUI and ScannerService, and handles interactions with backend APIs based on classification events.

This separation ensures that your ML logic is reusable, testable, and independent of your chosen UI framework.
